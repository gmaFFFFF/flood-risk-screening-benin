### Расчет в ArcGis
* Вычислить [[SCS-CN|Soil Conservation Service Curve Number]]
    * Объединить растры [[Карта землепользования и покрытий|покрытий и грунтов]] и водопроницаемости почв по [[Hydrologic Soil Group]] с помощью инструмента [Combine (Spatial Analyst)](https://pro.arcgis.com/ru/pro-app/3.6/tool-reference/spatial-analyst/combine.htm)
    * Добавить новое поле «CN» (Номер кривой) в таблицу атрибутов растров (RAT)
    * Рассчитать значение поля «CN» по таблицам [[SCS-CN#Определение CN]] с помощью выражения на Python
        * Пример для [[SCS-CN#Удовлетворительное гидрологическое состояние#AMC II (Норма)]]:

> [!bug]
> Таблица сверена с [[SCS-CN#CN для Карта землепользования и покрытий Карта земного покрова по данным r.curvenumber]] и выявлено расхождение в оценке пашни. Принято, что на пашне сток всё-таки выше.

Выражение:
``` Python
get_cn(!WorldCover_Mozaic!,!HiHydroSoil!)
```
Блок кода:
``` Python
def get_cn(land_class, soil_group):
    
    # Группы почвы: 1=A, 2=B, 3=C, 4=D
    # Двойные группы: 14,24 условно C, 34 условно D
    doouble_soil_group = {14: 3, 24: 3, 34: 4}
    soil_group = doouble_soil_group.get(soil_group, soil_group)
    
    # Словарь: Ключ = (Класс покрытия, Группа почвы), значение = CN
    lookup = {
        (10, 1): 36, (10, 2): 60, (10, 3): 73, (10, 4): 79,   # Деревья
        (20, 1): 55, (20, 2): 72, (20, 3): 81, (20, 4): 86,   # Кустарники
        (30, 1): 49, (30, 2): 69, (30, 3): 79, (30, 4): 84,   # Луг
        (40, 1): 70, (40, 2): 80, (40, 3): 87, (40, 4): 90,   # Пашня по r.curvenumber
        (50, 1): 89, (50, 2): 92, (50, 3): 94, (50, 4): 95,   # Застройка
        (60, 1): 65, (60, 2): 79, (60, 3): 87, (60, 4): 90,   # Грунт
        (70, 1): 100,  (70, 2): 100,  (70, 3): 100,  (70, 4): 100,    # Снег
        (80, 1): 100,(80, 2): 100,(80, 3): 100,(80, 4): 100,  # Вода
        (90, 1): 80, (90, 2): 80, (90, 3): 80, (90, 4): 80,   # Затапливаемые
        (95, 1): 100,  (95, 2): 100,  (95, 3): 100,  (95, 4): 100,    # Мангры. Нули превращены в 100, чтобы избежать деления на 0
        (100, 1): 74,(100, 2): 77,(100, 3): 78,(100, 4): 79   # Мхи
    }
    
    # Возвращаем значение из словаря, если ключа нет - возвращаем консервативные 100
    return lookup.get((land_class, soil_group), 100)
```

>[!Info]
>По манграм и снегу 0 значения стока заменены на консервативные 100 чтобы исключить деление на 0

* Создать новый растр, в котором CN станет полем `Value`
    * Инструмент «Калькулятор растров» по умолчанию работает только со значением `Value`.
    * Поэтому преобразуем растр с помощью инструмента [Справочная перекодировка](https://pro.arcgis.com/ru/pro-app/3.6/tool-reference/3d-analyst/lookup.htm)


* Вычислить сток с ячейки с помощью [Калькулятора растров](https://pro.arcgis.com/ru/pro-app/3.6/tool-reference/image-analyst/raster-calculator.htm) на основе [[SCS-CN#Формулы]]
``` Python
Con("SCS_CN" == 100, "Ari5day",
    Con("Ari5day" > 0.05 * (25400 / "SCS_CN" - 254), 
        Power("Ari5day" - 0.05 * (25400 / "SCS_CN" - 254), 2) / ("Ari5day" + 0.95 * (25400 / "SCS_CN" - 254)), 
        0)
   )
```
