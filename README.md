## Новизна / Novelty

В качестве альтернативы стандартному евклидову расстоянию от водотоков, применяемому при анализе факторов затопления, в работе апробирован метод расчета **энергетической стоимости распространения водного потока** от русла к водоразделу.

### Энергетическое расстояние до водотоков

Метод рассчитывает **кумулятивные потери напора паводка** (в метрах) при гипотетическом движении воды от русла к водоразделу с помощью инструмента ArcGIS «[Накопление расстояния](https://pro.arcgis.com/ru/pro-app/3.6/tool-reference/spatial-analyst/distance-accumulation.htm)».

Подробнее ознакомиться с физико-математической базой алгоритма можно в заметке [Энергетическое расстояние до водотоков](10_research_notes/гис/алгоритмы/Расстояние%20до%20водотоков.md).

Аналогичные подходы к расчету расстояния до гидрографической сети в доступных автору публикациях не встречались. Автор ожидает, что фактор энергетического расстояния может стать одним из ключевых предикатов при региональном скрининге паводковых рисков.

#### Сравнение с HAND
В отличие от классического HAND (Height Above the Nearest Drainage), который учитывает только относительную высоту поверхности, данный метод эвристически интегрирует уклон, наклонное расстояние и шероховатость поверхности (коэффициент Маннинга). Это повышает его дифференцирующую способность на равнинных территориях, где HAND часто дает излишне генерализованные результаты.

#### Текущий статус и ограничения

Метод носит **эвристический характер**. Из-за архитектурных ограничений инструмента «Накопление расстояния» алгоритм использует компромиссный базовый гидравлический уклон ($i_{base} \approx 4‰$) в качестве константы вместо индивидуального расчета для каждого пикселя.

Корректность модели предварительно проверена через согласованность (кросс-валидацию) с результатами HAND и независимыми модельными данными затопления. Верификация на полевых данных не проводилась.

Существует вероятность, что метод может занижать риски для удаленных, но низкорасположенных участков, поэтому на данном этапе его рекомендуется применять **в связке с HAND** для взаимной компенсации ограничений.

Откройте репозиторий как Obsidian Vault. Откройте [план исследования](20_project_gis/plan_research.md) для изучения заметок в логической последовательности их создания автором.

### Путь проектного менеджмента (трудоемкость и эффективность)
Для оценки реальных трудозатрат на проведение пространственного анализа и ГИС-моделирования откройте файл **[research_time_log.xlsx](research_time_log.xlsx)**. В нем зафиксирован хронометраж всех этапов работы над проектом.

> [!Note] 
> В хронометраж включен столбец **«Длительность полезная, час»**.
> Эта метрика отражает продуктивную работу, т.е. из нее исключены затраты времени, которые, по мнению автора, можно было не делать: исправление ошибок, движение по ложному пути, слишком долгое изучение документации и т.п.
> Метрика позволяет оценить чистую трудоемкость воссоздания модели без учета квалификации автора.
