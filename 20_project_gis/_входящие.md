# Расчет времени бассейнового добегания

Урок: https://courses.gisopencourseware.org/mod/book/view.php?id=1371&chapterid=3380

1. Найти максимальное расстояние от самой удаленной точки водосбора до его устья.
    1. **Инструмент `Flow Length`**[](https://pro.arcgis.com/en/pro-app/3.4/tool-reference/spatial-analyst/flow-length.htm). Установите параметр **`direction_measurement`** в значение **`UPSTREAM`**. Это даст для каждой ячейки расстояние до истока (раздела). Для получения L по каждому водосбору используйте `Zonal Statistics`, чтобы найти **максимальное** значение длины потока в пределах каждого полигона водосбора[](https://pro.arcgis.com/en/pro-app/3.4/tool-reference/spatial-analyst/flow-length.htm).
2. Определить средний уклон вдоль самого длинного пути.
    1.  **Определите путь:** Конвертируйте ячейку с `Max_FlowLen` в точку. Используйте `Flow Path` (или `Flow Accumulation` с условием) для построения линии пути от этой точки вниз по течению к устью[](https://courses.gisopencourseware.org/mod/book/view.php?id=1371).  
    2. **Вычислите уклон:** Используйте `Zonal Statistics` вдоль построенной линии (в качестве зоны) для получения **среднего** значения уклона из ЦМР или растра уклонов.
3. Рассчитать t_c для каждого водосбора.
    * Добавьте в таблицу атрибутов водосборов новые поля и вычислите их с помощью **Калькулятора полей**.  
    *  **Разность высот (H):** `H = L * S`.  
    * **Формула Кирпича (в минутах):** `t_c = 0.0195 * ( (L^0.77) / ( (H/L)^0.385 ) )`[](https://courses.gisopencourseware.org/mod/book/view.php?id=1371). Для преобразования в часы: `t_c_hours = t_c / 60`.
    * Альтернатива: - **SCS Lag Equation** широко используется в гидрологическом моделировании (например, в HEC-HMS) для бассейнов площадью до 800 га: 
        1.  Склоновый поток: `t_lag = (L^0.8 * [(25400/CN) - 254 +25.4]^0.7) / (4238 * S^0.5)`, где `CN` — среднее значение кривой числа для водосбора[](https://docs.bentley.com/LiveContent/web/Drainage%20and%20Utilities%20CONNECT%20Edition%20Help-v5/en/60005.html). Расчет `L` и `S` аналогичен.
        2. Связь с временем концентарции `t_lag = t_c * 0.6`
    * формула кинематического решения Маннинга (**Overton and Meadows, 1976**):
    * Для средних бассейнов:
        * Формула Брансби-Вильямса (Bransby Williams)
        * Формула Джандотти (Giandotti)
    * Для любых бассейнов:
        * Суммирование времени движения воды по самому длинному гидравлическому пути $t_c = \sum\frac{L_i}{v_i}$

Пример скрипта: 
``` Python
import arcpy
from arcpy.sa import *
from arcpy import env

watershed = # Путь к полигону водосбора
dem = # Путь к цифровой модели рельефа
fdr = # Путь к растру направления стока
env.workspace = # Путь к рабочей папке

intermediate = []  # Промежуточные данные

try:
    arcpy.SetProgressorLabel("Clipping flow direction...")
    fdrClip = ExtractByMask(fdr, watershed) # Обрезка растра направлений стока по границам водосбора
    intermediate.append(fdrClip)

    arcpy.SetProgressorLabel("Computing longest flow path for the basin...")
    flowLength = Int(FlowLength(fdrClip, "DOWNSTREAM")) # Расчет расстояния по направлению стока ("DOWNSTREAM") от каждой ячейки до выхода
    arcpy.CalculateStatistics_management(flowLength) 
    maxValue = int(arcpy.GetRasterProperties_management(flowLength, "MAXIMUM").getOutput(0)) #- Нахождение максимального значения длины стока (самой удаленной точки от выхода).
    maxCell = Con(flowLength == maxValue, 1) # Создание растра, где только одна ячейка (с максимальной длиной) имеет значение 1
    intermediate.append(maxCell)

    # Использование обратного направления потока? 
    # Но в данном коде используется dem как стоимость, что может быть неверно.
    # Возможно, следует использовать инвертированный dem или другой растр стоимости.
    lfpRas = CostPath(maxCell, dem, fdrClip, destination_field="Value") # Построение оптимального пути от найденной точки до выхода.
    intermediate.append(lfpRas)

    rasLine = arcpy.RasterToPolyline_conversion(lfpRas, "rasLine", "ZERO", simplify="NO_SIMPLIFY") # Преобразование растрового пути в векторную линию.
    intermediate.append(rasLine)

    lfp = arcpy.Dissolve_management(rasLine, "LongestFlowPath") # Объединение (dissolve) всех сегментов в единую линию
    intermediate.append(lfp)

finally:
    arcpy.SetProgressorLabel("Removing Intermediate data... ")
    # Все промежуточные данные сохраняются в списке `intermediate` и удаляются после завершения работы
    for data in intermediate:
        arcpy.Delete_management(data)
    del intermediate
```
