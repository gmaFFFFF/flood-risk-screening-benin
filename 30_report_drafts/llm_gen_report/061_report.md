#### Расчет основных водотоков {#sec:117}

Как было показано в предыдущем разделе (раздел -@sec:116), полученные растровые слои стока служат основой для выделения гидросети. Для обеспечения физической корректности последующих гидрологических расчетов необходимо наличие сети основных водотоков, строго согласованной с используемой цифровой модельрельефа (ЦМР).

Первоначально рассматривалась возможность интеграции внешних векторных данных о водотоках из базы HydroATLAS (уровень детализации 10) [@linke2019]. Однако попытка их гармонизации с откорректированной ЦМР TanDEM-X выявила ряд непреодолимых проблем. Метод «прожига» (burning) тальвегов оказался неэффективным из-за геометрических нестыковок между независимыми наборами данных. Столкновение двух различных геометрий — векторных линий водотоков и растровых высот рельефа — приводило к возникновению значительных топологических артефактов, искажающих гидрологическую картину.

В связи с этим был осуществлен отказ от внешних источников в пользу расчета гидросети «с нуля» непосредственно на основе ЦМР и параметров поверхностного стока. Расчет водотоков из той же модели высот, на которой базируется вся методика, является не вынужденным компромиссом, а методологически верным решением, обеспечивающим замкнутость системы «ЦМР → гидросеть → факторы затопления». Во-первых, тальвег рассматривается как линия наименьших высот, и извлеченная таким образом гидросеть по определению строго соответствует поверхности, по которой будет моделироваться течение воды. Во-вторых, достигается внутренняя согласованность модели: неизбежные погрешности ЦМР одинаково распространяются на все производные слои, что полностью исключает возникновение топологических противоречий между рельефом и руслами водотоков.

Алгоритм выделения водотоков включал маршрутизацию и накопление поверхностного стока. Для расчета направления стока применялся классический алгоритм D8. Накопление стока выполнялось на основе предварительно рассчитанного растра максимальных осадков за 5 дней (Rx5day) для имитации условий паводка с периодом повторяемости 40 лет (ARI=40). Ключевым этапом алгоритма стало определение порога руслоформирования — минимальной площади водосбора, необходимой для образования постоянного водотока. Для эффективного поиска оптимального значения без полного перебора использовалась разреженная логарифмическая последовательность на основе чисел Маркова. В результате анализа принятое значение порога составило 7561 ячейку, что обеспечивает адекватное представление основных водотоков Республики Бенин при текущем пространственном разрешении ЦМР. Адекватность проведенной генерализации подтверждена экспертной верификацией полученной гидросети по картографической подложке.

Результатом работы алгоритма стала топологически корректная сеть основных водотоков представленная ранее на рисунке -@fig:basin.

Карта на рисунке -@fig:basin демонстрирует высокую степень детализации речной сети, при которой расчетные русла точно следуют линиям тальвегов. Это подтверждает правильность выбора порога накопления и позволяет использовать полученную сеть как достоверный базис для моделирования факторов удаленности.

Полученная гидросеть служит надежной основой для верификации всей модели поверхностного стока (раздел -@sec:118) и последующего гидрологического моделирования (раздел -@sec:119).
