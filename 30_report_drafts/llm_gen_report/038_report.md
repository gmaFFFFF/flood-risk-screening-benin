## Методика исследования {#sec:72}

В данном разделе разработанный теоретический и методологический аппарат преобразуется в практическую методику, адаптированную для условий ограниченной информационной базы Западной Африки. Сначала будет обоснована двухэтапная иерархическая структура подхода и её преимущества перед альтернативами, затем уточнено определение риска применительно к каждому этапу, и, наконец, введена система оценочных шкал, обеспечивающая сопоставимость факторов при их агрегации.

## Предлагаемый двухэтапный иерархический подход {#sec:73}

Двухэтапная структура выступает необходимым компромиссом между пространственным охватом и физической детальностью исследования. В условиях дефицита данных детальное гидродинамическое моделирование всей территории Республики Бенин технически не реализуемо и вычислительно нецелесообразно. В связи с этим первый этап — региональный геоинформационный скрининг — выполняет функцию пространственного фильтра, сужая область дальнейшего поиска исключительно до критических зон. Второй этап — детальное моделирование — обеспечивает физически обоснованную верификацию риска и расчёт конкретных параметров паводка непосредственно в выявленных зонах интереса.

**Этап I. Региональный геоинформационный скрининг.** Целью данного этапа является быстрая идентификация территорий с потенциально высокой опасностью затопления. Для её достижения решается ряд последовательных подзадач:
1. Расчёт ключевых факторов затопления.
2. Экспертное взвешивание факторов с помощью метода анализа иерархий (AHP).
3. Расчёт интегральной карты опасности.
4. Наложение фактора подверженности (плотность населения).
5. Выделение топ-5 наиболее критических зон с использованием алгоритма «Найти регионы» (Locate Regions).

Скрининг оперирует растровыми индексами и экспертными весами, что позволяет проводить оценку без численного решения дифференциальных уравнений гидродинамики.

**Этап II. Детальное моделирование критических зон (Fine-scale modeling).** Данный этап предназначен для уточнения параметров затопления, необходимых для принятия проектных и управленческих решений. В рамках этапа осуществляется построение двумерной (2D) гидродинамической модели для выбранных на предыдущем шаге участков. Производится симуляция сценариев паводка на основе уравнений мелкой воды (SWE) с использованием данных высокого разрешения, что позволяет выявить локальные «горячие точки» (hot spots) и разработать адресные защитные рекомендации.

Анализ блок-схемы подтверждает иерархическую связность этапов, где результат скрининга служит входным параметром для детальной симуляции. Таким образом, предложенный иерархический подход позволяет трансформировать неопределенность регионального масштаба в конкретные рекомендации для наиболее уязвимых территорий.

Выбор данной структуры требует детального сравнения с альтернативными методами.
